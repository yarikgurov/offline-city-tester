## 3. Найденные баги

### Баг #1: Утечка ID игрока в localStorage
- **Приоритет**: Низкий
- **Описание**: ID игрока хранится в localStorage без шифрования
- **Влияние**: Минимальное для демо-игры
- **Статус**: Не исправлено (не критично)

### Баг #2: Сброс данных при перезапуске сервера
- **Приоритет**: Средний
- **Описание**: Состояние игроков хранится в памяти сервера. При перезапуске все данные теряются
- **Воспроизведение**: Нажать Ctrl+C → node server.js
- **Рекомендация**: Сохранять состояние в файл gameState.json периодически
- **Статус**: Открыт

### Баг #3: Неполная валидация на сервере
- **Приоритет**: Низкий
- **Описание**: Теоретически можно отправить некорректные данные через прямой API-запрос
- **Статус**: Сервер возвращает ошибку в большинстве случаев

### Баг #4: Огонёк подсказки в смене 4 виден только при фокусе
- **Приоритет**: Низкий
- **Описание**: В смене «Сопоставление» огонёк показывается при наведении на select, а не постоянно
- **Статус**: Исправлено заменой select на кнопки

## 4. Предложения по сюжету

### 4.1 Сюжетная арка (реализовано)
Сюжет выстроен в связную историю:
1. Портал падает → восстанавливаем индикатор
2. Находим лишний лог → отсеиваем ложный след
3. Восстанавливаем последовательность запросов
4. Сопоставляем улики с проблемами
5. Взламываем повреждённый Service Worker
6. Ранжируем улики по важности
7. Находим аномалии в логах
8. Выявляем подмену ответов API
9. Восстанавливаем хронологию MITM-атаки
10. Выбираем протокол для защиты

### 4.2 Дополнительные концовки (реализовано)
4 ранга в зависимости от доверия и улик:
- 🏆 Мастер-детектив (доверие ≥ 80%, улик ≥ 8)
- 🔍 Опытный аналитик (доверие ≥ 50%, улик ≥ 5)
- ⚠️ Начинающий следователь (доверие ≥ 20%)
- 💀 Провал (доверие < 20%)

### 4.3 Предложения по улучшению
- Звуковое сопровождение (ambient cyberpunk)
- Анимированные переходы между сменами
- Кат-сцены с диалогами персонажей
- Система достижений
- Таймер на каждую смену (режим хардкора)
